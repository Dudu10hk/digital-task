# ✅ רשימת בדיקה להתקנה - Installation Checklist

השתמש ברשימה זו כדי לוודא שכל השלבים בוצעו.

---

## 📋 **לפני ההתחלה**

- [ ] Node.js 18+ מותקן (בדיקה: `node --version`)
- [ ] npm מותקן (בדיקה: `npm --version`)
- [ ] יש לך חשבון Supabase (חינמי)
- [ ] פרויקט Supabase נוצר
- [ ] יש לך 15-30 דקות זמן

---

## 🔧 **שלב 1: הורדה והתקנת תלויות**

- [ ] הפרויקט הורד/שובט למחשב
- [ ] פתחת טרמינל בתיקיית הפרויקט
- [ ] הרצת: `npm install` (או `pnpm install`)
- [ ] ההתקנה הסתיימה בהצלחה (ללא שגיאות)
- [ ] תיקיית `node_modules` נוצרה

---

## 🗄️ **שלב 2: הגדרת Supabase**

### 2.1 פרטי API

- [ ] נכנסת ל-Supabase Dashboard
- [ ] לחצת על Settings > API
- [ ] העתקת את ה-**Project URL**
- [ ] העתקת את ה-**anon public key**

### 2.2 קובץ משתני סביבה

- [ ] יצרת קובץ `.env.local` בתיקייה הראשית
- [ ] הוספת את `NEXT_PUBLIC_SUPABASE_URL=...`
- [ ] הוספת את `NEXT_PUBLIC_SUPABASE_ANON_KEY=...`
- [ ] הערכים תקינים (ללא "your-url" או "your-key")
- [ ] שמרת את הקובץ

### 2.3 סקריפטי SQL - קריטי!

עבור לSupabase Dashboard > SQL Editor והרץ את הקבצים הבאים **לפי הסדר**:

- [ ] `scripts/001_create_tables.sql` - טבלאות משתמשים
  - [ ] הרצת את הסקריפט
  - [ ] אין שגיאות
  - [ ] הצלחה: "Success"

- [ ] `scripts/002_create_task_tables.sql` - טבלאות משימות
  - [ ] הרצת את הסקריפט
  - [ ] אין שגיאות
  - [ ] הצלחה: "Success"

- [ ] `scripts/004_create_otp_table.sql` - טבלת OTP
  - [ ] הרצת את הסקריפט
  - [ ] אין שגיאות
  - [ ] הצלחה: "Success"

- [ ] `scripts/004_setup_storage.sql` - Storage לתמונות
  - [ ] הרצת את הסקריפט
  - [ ] אין שגיאות
  - [ ] הצלחה: "Success"

- [ ] `scripts/005_convert_to_jsonb.sql` - המרה ל-JSONB (קריטי!)
  - [ ] הרצת את הסקריפט
  - [ ] אין שגיאות
  - [ ] הצלחה: "Success"
  - [ ] ⚠️ זהו הסקריפט החשוב ביותר!

- [ ] (אופציונלי) `scripts/003_seed_demo_data.sql` - נתוני דמו
  - [ ] הרצת רק אם רצית נתונים לדוגמה

### 2.4 בדיקת Storage

- [ ] בדשבורד: Storage > Buckets
- [ ] רואה bucket בשם `profile-images`
- [ ] ה-bucket מוגדר כ-Public

---

## 👤 **שלב 3: יצירת משתמש Admin**

### דרך Supabase Dashboard (מומלץ):

- [ ] נכנסת ל-Authentication > Users
- [ ] לחצת "Add user" > "Create new user"
- [ ] מילאת Email: `admin@example.com`
- [ ] מילאת Password: `Admin123!` (או אחר)
- [ ] סימנת: "Auto Confirm User" ✅
- [ ] לחצת "Create user"
- [ ] המשתמש נוצר בהצלחה
- [ ] לחצת על המשתמש ברשימה
- [ ] הוספת ל-User Metadata:
  ```json
  {
    "name": "מנהל מערכת",
    "role": "admin",
    "avatar": ""
  }
  ```
- [ ] שמרת את השינויים
- [ ] המשתמש מאומת (✅ ליד Email)

## 🚀 **שלב 4: הרצת הפרויקט**

- [ ] פתחת טרמינל בתיקיית הפרויקט
- [ ] הרצת: `npm run dev`
- [ ] השרת עלה בהצלחה
- [ ] רואה: "Local: http://localhost:3000"
- [ ] אין שגיאות אדומות בטרמינל

## ✅ **שלב 5: בדיקה ראשונית**

### 5.1 כניסה למערכת

- [ ] פתחת דפדפן: http://localhost:3000
- [ ] רואה דף התחברות
- [ ] לחצת על טאב "סיסמה"
- [ ] הכנסת Email: `admin@example.com`
- [ ] הכנסת Password: `Admin123!`
- [ ] לחצת "התחבר"
- [ ] התחברת בהצלחה!
- [ ] רואה את לוח המשימות

### 5.2 בדיקת תכונות בסיסיות

- [ ] רואה 3 עמודות: To Do, In Progress, Done
- [ ] כפתור "משימה חדשה" קיים
- [ ] תפריט משתמש (למעלה מימין) עובד
- [ ] שם המשתמש מופיע: "מנהל מערכת"
- [ ] אין שגיאות בקונסול (F12)

### 5.3 יצירת משימה

- [ ] לחצת "משימה חדשה"
- [ ] דיאלוג נפתח
- [ ] מילאת כותרת: "בדיקת מערכת"
- [ ] מילאת תיאור: "משימת בדיקה ראשונה"
- [ ] לחצת "צור משימה"
- [ ] המשימה נוצרה בעמודת To Do
- [ ] המשימה מופיעה בלוח

### 5.4 בדיקת שמירה

- [ ] רענון דף (F5)
- [ ] המערכת ביקשה להתחבר שוב (נכון)
- [ ] התחברת שוב
- [ ] המשימה שיצרת עדיין קיימת ✅

## 🐛 **פתרון בעיות**

אם נתקלת בבעיה, סמן את השגיאה וקרא את הפתרון:

### שגיאות נפוצות:

- [ ] "Invalid API key" → בדוק `.env.local`, הפעל מחדש `npm run dev`
- [ ] "relation does not exist" → הרץ שוב את סקריפטי ה-SQL
- [ ] "Error saving tasks" → הרץ `005_convert_to_jsonb.sql`
- [ ] לא מצליח להתחבר → בדוק שהמשתמש קיים ומאומת
- [ ] תמונות לא נטענות → הרץ `004_setup_storage.sql`
